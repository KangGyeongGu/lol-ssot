## 0. 문서 목적
- 공통 작업(커밋/푸시 등) 전용 에이전트의 범위와 경계를 정의한다.
- 도메인 에이전트와의 충돌을 최소화한다.

## 관련 문서
- `AGENTS/CLAUDE/README.md`
- `README.md`

---
## 1. 분할 원칙
- 공통 작업만 담당하며 도메인 변경에는 관여하지 않는다.
- 실제 코드/문서 수정은 도메인 에이전트가 수행한다.

---
## 2. 운영 에이전트
- Commit Agent: `COMMIT_AGENT.md`

---
## 3. 공통 작업 범위 표
| 영역 | Agent | 대상 |
|---|---|---|
| Commit/Push | `COMMIT_AGENT.md` | 레포지토리 전체 변경 사항 |

---
## 4. 경계 규칙
- Commit Agent는 변경 내용을 생성하지 않고 커밋/푸시만 수행한다.
- 커밋 메시지에 AI 서명(Co-authored-by)을 추가하지 않는다.
