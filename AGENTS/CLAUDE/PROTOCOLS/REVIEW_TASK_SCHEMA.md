# Review Task Schema

Purpose
- Standardize consolidated task cards generated by review masters.

Required Fields
- ID: `TASK-S-XXX` or `TASK-Q-XXX`
- TYPE: `SSOT | QUALITY`
- DOMAIN: domain key
- PRIORITY: `Critical | High | Medium | Low`
- TITLE: concise task title
- SCOPE: what to change
- ACCEPTANCE: done criteria
- DEPENDENCIES: `none` or task IDs
- SOURCE_FINDINGS: source raw finding IDs
- RELATED_REPORTS: source raw report filenames

Guideline
- If SSOT and QUALITY findings share a root cause, keep SSOT task primary and link QUALITY task through DEPENDENCIES or SOURCE_FINDINGS.
